## 一、产品定位

### 1.1 演进路径

| 阶段 | 定位 | 核心功能 | 目标用户 | 时间 |
|------|------|----------|----------|------|
| **Phase 1** | 学习项目 | 单用户叫车流程 | 自己 | 1-2 周 |
| **Phase 2** | 团队工具 | 多用户 + 费用管理 | 小团队 | 4-6 周 |
| **Phase 3** | 企业平台 | 批量派单 + 监控分析 | 企业客户 | 8-12 周 |

### 1.2 Phase 1 范围（当前目标）

**必须做（P0）：**
- [x] 地图定位与显示
- [ ] 地址搜索（起点/终点）
- [ ] 车型选择（快车/专车）
- [ ] 费用预估
- [ ] 叫车按钮 + 订单状态
- [ ] 司机信息展示
- [ ] 行程实时追踪

**可选（P1）：**
- [ ] 系统托盘常驻
- [ ] 深色模式
- [ ] 历史订单
- [ ] 支付模拟

**不做（Phase 2+）：**
- 真实支付集成
- 多用户管理
- 企业审批流程
- 批量派单

## 四、开发计划

### 4.1 Phase 1 任务分解

| 模块 | 任务 | 预估时间 |
|------|------|----------|
| **项目初始化** | Electron + Vite + React 搭建 | 2h |
| **主进程** | 窗口、托盘、IPC | 4h |
| **地图模块** | 高德地图集成 | 4h |
| **叫车流程** | 选点、车型、估价 | 6h |
| **订单系统** | 状态机、实时更新 | 6h |
| **用户系统** | 登录、Token | 4h |
| **UI 完善** | 样式、动画 | 4h |
| **测试** | 功能测试 | 4h |
| **总计** | | **~34h (1-2周)** |

### 4.2 并行开发策略

使用 **4 个 Agent 并行开发**：

| Agent | 职责 | 文件范围 |
|-------|------|----------|
| Agent 1 | 主进程架构 | `electron/` |
| Agent 2 | 地图模块 | `src/components/Map/` |
| Agent 3 | 订单系统 | `src/components/Order/` |
| Agent 4 | 用户系统 | `src/components/User/` |

## 五、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 滴滴 API 不可用 | 高 | 使用 Mock API 开发 |
| 高德地图限制 | 中 | 申请免费 Key，或用 Leaflet 备选 |
| 实时定位精度 | 中 | 使用 IP 定位作为备选 |
| 支付安全 | 高 | Phase 1 仅模拟，不真实支付 |
