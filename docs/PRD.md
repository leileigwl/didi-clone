# 滴滴打车桌面应用 PRD (产品需求文档)

## 1. 产品概述

### 1.1 产品名称
DiDi Desktop - 滴滴打车桌面客户端

### 1.2 产品定位
一款基于 Electron 的跨平台桌面打车应用，提供与滴滴打车移动端一致的用户体验。

### 1.3 目标用户
- 需要在办公电脑上直接叫车的白领用户
- 偏好大屏幕操作的老年用户
- 需要批量管理用车企业的管理员

## 2. 核心功能模块

### 2.1 用户系统
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 手机号登录 | 短信验证码登录 | P0 |
| 微信登录 | 扫码授权登录 | P1 |
| 个人中心 | 用户信息、行程记录、钱包 | P0 |
| 实名认证 | 身份证认证（打车必需） | P0 |

### 2.2 叫车流程
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 地图展示 | 显示当前位置和周边车辆 | P0 |
| 地址搜索 | POI搜索、历史记录、收藏 | P0 |
| 路线规划 | 起点-终点路线预览 | P0 |
| 车型选择 | 快车、专车、出租车等 | P0 |
| 估价展示 | 预估价格和到达时间 | P0 |
| 发单叫车 | 提交订单等待接单 | P0 |
| 实时调度 | 显示司机接单、位置、预计到达 | P0 |

### 2.3 行程中
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 实时轨迹 | 司机行驶路线实时更新 | P0 |
| 司机信息 | 头像、姓名、车牌、电话 | P0 |
| 一键通话 | 加密电话联系司机 | P1 |
| 分享行程 | 分享给亲友实时位置 | P1 |
| 安全中心 | 紧急联系人、一键报警 | P0 |

### 2.4 支付系统
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 在线支付 | 微信/支付宝支付 | P0 |
| 余额支付 | 滴滴钱包余额 | P1 |
| 发票管理 | 开具电子发票 | P2 |
| 优惠券 | 使用优惠券抵扣 | P1 |

### 2.5 订单管理
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 待接单 | 显示等待中的订单 | P0 |
| 进行中 | 显示行程中订单 | P0 |
| 历史订单 | 查看过往行程记录 | P0 |
| 订单详情 | 行程详情、费用明细 | P0 |
| 取消订单 | 取消叫车（含取消费用） | P0 |
| 投诉反馈 | 对行程进行评价投诉 | P1 |

## 3. 技术架构

```
┌─────────────────────────────────────────────────┐
│                   Electron App                   │
├─────────────────────────────────────────────────┤
│  Main Process          │    Renderer Process    │
```

## 5. API 接口设计

### 5.1 认证模块
```yaml
POST /api/auth/sms/send
  请求: { phone: "13800138000" }
  响应: { code: 0, message: "发送成功" }

POST /api/auth/login
  请求: { phone: "13800138000", code: "123456" }
  响应: { token: "xxx", user: {...} }
```

### 5.2 订单模块
```yaml
POST /api/order/create
  请求: {
    startLocation: { lat, lng, name },
    endLocation: { lat, lng, name },
    carType: "express"
  }
  响应: { orderId: "xxx", estimatedPrice: 28 }

GET /api/order/:id
  响应: { status, driver, currentLocation, ... }

POST /api/order/:id/cancel
  响应: { refundAmount: 0 }
```

### 5.3 WebSocket 事件
```yaml
# 客户端订阅
subscribe: { event: "order:status", orderId: "xxx" }

# 服务端推送
order:accepted   # 司机接单
order:arriving   # 司机到达
order:started    # 行程开始
order:completed  # 行程结束
driver:location  # 司机位置更新
```

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 地图 API 限制 | 高 | 使用高德/百度地图 Web API |
| 实时定位精度 | 中 | 结合 IP 定位作为备选 |
| 支付安全 | 高 | 不存储支付信息，跳转网页支付 |
| 滴滴 API 不可用 | 高 | 模拟数据开发，后续对接真实 API |

## 8. 成功指标

- 首屏加载时间 < 3s
- 内存占用 < 200MB
- 地图渲染流畅 (60fps)
- 订单状态延迟 < 1s
- 应用崩溃率 < 0.1%
